# Rate My Politician — Requirements

> **Civic accountability dApp for Nepal.**
> Citizens rate politicians on parliamentary performance and integrity.
> Verified by Janamat zkID. Immutable on Solana.

---

## Scaffold command

```bash
npx create-solana-dapp@latest rate-my-politician --template nextjs-anchor
cd rate-my-politician
```

This gives you **Next.js 16 + Tailwind v4 + `@solana/kit` + Anchor** out of the box.
Everything below is built on top of that scaffold — never replace or fight the template conventions.

## Anchor program — what to build (`anchor/programs/rate-my-politician/src/lib.rs`)

Replace the vault program entirely. Three instructions:

### Instructions

**`initialize_politician(politician_id: String)`**
- Called once per politician by the deployer/admin
- Creates a `PoliticianAccount` PDA
- Seeds: `["politician", politician_id.as_bytes()]`

**`submit_rating(politician_id, integrity, work_ethic, promises_kept, overall, zk_verified)`**
- Called by a voter (connected wallet)
- Creates a `RatingAccount` PDA — seeds enforce 1 rating per (politician × voter)
- Seeds: `["rating", politician_id.as_bytes(), voter.key().as_ref()]`
- Updates `PoliticianAccount` aggregate sums atomically
- `zk_verified: bool` — set `true` when Janamat zkID proof is validated

**`update_rating(integrity, work_ethic, promises_kept, overall)`**
- Lets voter revise their rating within 24h of submission
- Adjusts aggregate: subtracts old values, adds new values

### Accounts

**`PoliticianAccount`** — one per politician, persistent
```
politician_id:      String   (max 64 chars)
authority:          Pubkey   (deployer)
total_ratings:      u64
integrity_sum:      u64
work_ethic_sum:     u64
promises_kept_sum:  u64
overall_sum:        u64
bump:               u8
created_at:         i64
last_updated:       i64
```

**`RatingAccount`** — one per (politician × voter), permanent
```
politician_id:  String
voter:          Pubkey
integrity:      u8   (1–5)
work_ethic:     u8   (1–5)
promises_kept:  u8   (1–5)
overall:        u8   (1–5)
zk_verified:    bool
timestamp:      i64
bump:           u8
```

### Errors
- `InvalidRating` — value outside 1–5
- `IdTooLong` — politician_id > 64 chars
- `Overflow` — arithmetic overflow on sums
- `UpdateWindowExpired` — update attempted > 24h after submission

## `app/janamat/sentiment.ts`

Two functions:

**`classifyOptionSentiment(text: string): 'positive' | 'negative' | 'neutral'`**
Match against Nepali + English keyword lists. Positive: `सकारात्मक`, `सही`, `Strongly in favour` etc. Negative: `गलत`, `स्टन्ट`, `wrong`, `corrupt` etc.

**`computePollApproval(poll: JanamatPoll): number`**
Returns 0–100. `(positiveVotes / totalVotes) × 100`. Returns 50 if no votes.

**`computeJanamatScore(polls: JanamatPoll[]): number`**
Weighted average: `weight = log(totalVotes + 1) × exp(-daysOld / 30)`.
Returns 50 if no polls.

## `app/data/scoring.ts`

```
compositeScore (0–100) =
  (objectiveScore × 0.60) + (communityScore × 0.40)

objectiveScore =
  (attendancePercent × 0.40) + (billScore × 0.60)
  billScore = min(billsProposed × 5, 50) + min(passRate × 0.5, 50)

communityScore =
  (janamatScore × 0.50) + (starScore × 0.50)
  starScore = ((avg of 4 star categories) - 1) × 25   → 0–100

Grade: A ≥ 80 | B ≥ 65 | C ≥ 50 | D ≥ 35 | F < 35
```

## Key constraints

- **Use `@solana/kit`** — the template uses the new Solana kit, not the old `@solana/web3.js`. Don't mix them.
- **Use Codama-generated client** — after `anchor build`, always run `npm run codama:js` and import from `app/generated/`. Never hand-write instruction builders.
- **Use `useSendTransaction`** from the template — don't write your own transaction sending logic.
- **Use `useWallet` from `app/lib/wallet/context`** — not from `@solana/wallet-adapter-react`.
- **Use `sonner` for toasts** — already in the template, keep it consistent.
- **Tailwind v4** — no `tailwind.config.js`, uses CSS-first config in `globals.css`.
- **One rating per (voter × politician)** — enforced on-chain by PDA seed. The frontend should check before letting users open the form.
- **zkID is a flag, not a gate** — for hackathon MVP, `zk_verified = false` is fine. Wire the real Janamat SDK post-hackathon.
- **Janamat API is public, no rate limit found** — but add 300ms delays between paginated fetches in the scraper to be polite.
